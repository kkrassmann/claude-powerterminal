    **1. Create CodeReviewPanelComponent** — the fullscreen overlay container.

    Component structure:
    - Inputs: `sessionId: string`, `cwd: string`
    - Outputs: `close: EventEmitter<void>`
    - Inject `CodeReviewService`
    - On init: call `codeReviewService.fetchDiff(cwd)`, then parse with `diff2html.parse()` to get `DiffFile[]`
    - Apply `sortFilesByLayer()` from code-review.model.ts using `detectProjectType()` on the file list
    - Track `selectedFileIndex: number` (default 0), `outputFormat: 'side-by-side' | 'line-by-line'` (default 'side-by-side')
    - Prev/Next navigation: decrement/increment selectedFileIndex, bounds-checked
    - Close on Escape key: `@HostListener('document:keydown.escape')` calls `close.emit()`
    - Loading state while diff is being fetched
    - Error state if diff fetch fails

    Template layout:
    ```html
    <div class="review-overlay" (click)="close.emit()">
      <div class="review-container" (click)="$event.stopPropagation()">
        <div class="review-header">
          <h2>Code Review</h2>
          <div class="review-controls">
            <button (click)="toggleFormat()">{{ outputFormat === 'side-by-side' ? 'Unified' : 'Side-by-Side' }}</button>
            <span class="file-counter">{{ selectedFileIndex + 1 }} / {{ files.length }}</span>
            <button (click)="prevFile()" [disabled]="selectedFileIndex === 0">Prev</button>
            <button (click)="nextFile()" [disabled]="selectedFileIndex === files.length - 1">Next</button>
          </div>
          <button class="close-btn" (click)="close.emit()">&#x2715;</button>
        </div>
        <div class="review-body">
          <app-file-tree [files]="files" [selectedIndex]="selectedFileIndex" (fileSelected)="onFileSelected($event)"></app-file-tree>
          <app-diff-viewer [file]="selectedFile" [outputFormat]="outputFormat" [rawDiff]="rawDiff"></app-diff-viewer>
        </div>
      </div>
    </div>
    ```

    CSS: Fixed position overlay at z-index 1001 (above session-detail's 1000). Background `rgba(0,0,0,0.7)`. Container: `background: #1e1e2e` (Catppuccin base), `border-radius: 8px`, `width: 95vw`, `height: 92vh`, centered. Header: flex row, `background: #181825` (mantle). Body: flex row, file-tree takes `width: 280px`, diff-viewer takes remaining space.

    **1. Create DiffViewerComponent** — renders a single file's diff using diff2html.

    Component structure:
    - Inputs: `file: DiffFile | null` (single parsed diff file), `outputFormat: 'side-by-side' | 'line-by-line'`, `rawDiff: string` (full raw diff for re-rendering)
    - Outputs: `lineClicked: EventEmitter<{ fileIndex: number, line: number, side: 'old' | 'new' }>` (for future comment placement)
    - ViewChild: `diffContainer` ElementRef for direct DOM manipulation
    - On changes (ngOnChanges): re-render the diff for the current file

    **Rendering approach — hybrid (per RESEARCH.md recommendation):**
    Use `diff2html.html()` to generate the HTML (handles complex word-level highlighting, line numbering, side-by-side alignment), then augment the DOM with click handlers.

    ```typescript
    import { html as renderDiffHtml, parse as parseDiff } from 'diff2html';
    import type { DiffFile } from 'diff2html/lib/types';
    ```

    When `file` input changes:
    1. If file is null, show empty state
    2. Build a single-file diff string from the DiffFile's blocks (reconstruct the unified diff header + hunks) OR re-parse the raw diff and filter to the selected file
    3. Render using `renderDiffHtml()` with options:
       - `outputFormat`: from input ('side-by-side' or 'line-by-line')
       - `drawFileList: false` (we have our own file tree)
       - `matching: 'lines'`
       - `diffStyle: 'word'` (word-level inline diffs per user decision)
       - `colorScheme: 'dark'`
    4. Set `this.diffContainer.nativeElement.innerHTML = html` (bypass Angular sanitizer — diff comes from git, trusted source)
    5. Add CSS class `d2h-dark-color-scheme` to the container
    6. Attach click handlers on line number elements (`.d2h-code-linenumber`) for future comment placement — emit `lineClicked` event with file and line info

    **New/added files:** If the file status is "Added" (oldName === '/dev/null'), show collapsed by default with a header showing filename + line count. Add a toggle button to expand/show the full diff. Per user decision: "New/added files shown collapsed by default with filename + line count header; click to expand."

    **2. Catppuccin Mocha CSS overrides** in `diff-viewer.component.css`:

    Override diff2html's dark mode CSS variables:

    ```css
      --d2h-dark-file-header-bg-color: #181825;

      --d2h-dark-empty-line-bg-color: #11111b;
      --d2h-dark-ins-bg-color: rgba(166, 227, 161, 0.15);
      --d2h-dark-ins-highlight-bg-color: rgba(166, 227, 161, 0.30);
      --d2h-dark-del-bg-color: rgba(243, 139, 168, 0.15);
      --d2h-dark-del-highlight-bg-color: rgba(243, 139, 168, 0.30);
      --d2h-dark-info-bg-color: #181825;
      --d2h-dark-info-color: #89b4fa;
    }
    ```

    Also override highlight.js classes for syntax highlighting colors (Catppuccin Mocha):
    - `.hljs-keyword`: #cba6f7 (mauve)
    - `.hljs-string`: #a6e3a1 (green)
    - `.hljs-number`: #fab387 (peach)
    - `.hljs-comment`: #6c7086 (overlay0)
    - `.hljs-function`, `.hljs-title`: #89b4fa (blue)
    - `.hljs-type`: #f9e2af (yellow)
    - `.hljs-built_in`: #f38ba8 (red)
    - `.hljs-attr`: #f9e2af (yellow)

    **3. Import diff2html CSS globally:**
    Add `@import 'diff2html/bundles/css/diff2html.min.css';` to the component CSS or, if Angular's view encapsulation blocks it, add the CSS path to `angular.json` styles array. The component CSS should use `::ng-deep` or `:host ::ng-deep` prefix for diff2html class overrides if needed, OR set `encapsulation: ViewEncapsulation.None` on the component (since it's a fullscreen overlay, style leakage is minimal and contained).

    **IMPORTANT:** Do NOT use Angular's `[innerHTML]` binding for the diff HTML. Angular's DomSanitizer strips CSS classes and data attributes from diff2html output. Instead, use `this.diffContainer.nativeElement.innerHTML = html` via ViewChild (per RESEARCH.md Pitfall 1).
