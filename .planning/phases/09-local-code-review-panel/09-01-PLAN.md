    **1. Add IPC channel constants** to `src/shared/ipc-channels.ts`:
    ```
    REVIEW_DIFF: 'review:diff'           // Get full unified diff for all uncommitted changes
    REVIEW_REJECT_HUNK: 'review:reject-hunk'  // Reject a single hunk via reverse patch
    REVIEW_REJECT_FILE: 'review:reject-file'  // Reject entire file (revert all changes)
    ```

    **2. Create shared type definitions** in `src/src/app/models/code-review.model.ts`:
    - `ReviewComment` interface: `id: string`, `sessionId: string`, `filename: string`, `line: number`, `text: string`, `resolved: boolean`, `createdAt: Date`
    - `ReviewFileStatus` type: `'added' | 'modified' | 'deleted' | 'renamed'`
    - `ReviewHunkState` type: `'pending' | 'accepted' | 'rejected'`
    - `ReviewFileState` interface: `filename: string`, `status: ReviewFileStatus`, `hunkStates: ReviewHunkState[]`, `reviewed: boolean`
    - `ProjectType` interface: `name: string`, `layerOrder: string[]` (for intelligent file ordering)
    - Export `PROJECT_TYPES` array with at least `angular` and `express-api` project types with their layer orders per CONTEXT.md decisions (angular: routes, components, services, models, shared, tests; express-api: routes, controllers, managers, brokers, models, middleware, tests)
    - Export `detectProjectType(changedFiles: string[]): string | null` function that detects project type from file patterns (has `angular.json` or `src/app/` pattern -> angular; has `routes/` + `controllers/` or `managers/` -> express-api; fallback null)
    - Export `sortFilesByLayer(files: { filename: string }[], projectTypeName: string | null): { filename: string }[]` function that sorts by layer order, fallback alphabetical

    **3. Create IPC handlers** in `electron/ipc/review-handlers.ts`:
    Follow the pattern from `git-handlers.ts` (use `execFile` with promisify, `windowsHide: true`, timeout).

    - `review:diff` handler: Takes `cwd: string`. Runs `git diff HEAD --unified=3` with `maxBuffer: 10 * 1024 * 1024` (10MB) and `timeout: 15000` (15s). Also runs `git diff --unified=3` (without HEAD, for staged-only case) and `git diff --cached --unified=3` (for staged changes). Combine: if `git diff HEAD` returns output, use that. This captures both staged and unstaged changes against HEAD. Returns the raw diff string.

    - `review:reject-hunk` handler: Takes `cwd: string`, `patchContent: string`. Uses `child_process.spawn` (NOT execFile — execFile doesn't support stdin) to run `git apply --reverse --unidiff-zero` with the patch content written to stdin. Returns `{ success: boolean, error?: string }`.

    - `review:reject-file` handler: Takes `cwd: string`, `filePath: string`. Runs `execFile('git', ['checkout', 'HEAD', '--', filePath])` with `timeout: 5000`. Returns `{ success: boolean, error?: string }`.

    Export `registerReviewHandlers()` function that registers all three handlers.

    **4. Register handlers in main.ts:**
    - Import `registerReviewHandlers` from `'./ipc/review-handlers'`
    - Call `registerReviewHandlers()` in the same block where other handlers are registered (after `registerAnalysisHandlers()`)

    **IMPORTANT for spawn-based stdin pipe:** The `review:reject-hunk` handler MUST use `spawn` from `child_process`, not `execFile`, because `execFile` does not support the `input` option. Write patchContent to `proc.stdin`, then call `proc.stdin.end()`. Collect stderr, resolve on `close` event.

    - `POST /api/review/reject-hunk` — Accepts JSON body `{ cwd: string, patchContent: string }`. Uses `spawn` with stdin for `git apply --reverse --unidiff-zero`. Returns `{ success: boolean, error?: string }`.

    - `POST /api/review/reject-file` — Accepts JSON body `{ cwd: string, filePath: string }`. Uses `execFileAsync('git', ['checkout', 'HEAD', '--', filePath])`. Returns `{ success: boolean, error?: string }`.

    Import `spawn` from `child_process` (add to existing imports). Reuse the existing `corsHeaders` and CORS preflight pattern. For POST endpoints, parse the JSON body the same way as the existing `POST /api/sessions` handler (req.on('data') + req.on('end')).

    **2. Create Angular CodeReviewService** in `src/src/app/services/code-review.service.ts`:

    Follow the dual-transport pattern from `log-analysis.service.ts` (check `window.electronAPI` for IPC, fallback to HTTP fetch).

    - `fetchDiff(cwd: string): Promise<string>` — IPC: `window.electronAPI.invoke(IPC_CHANNELS.REVIEW_DIFF, cwd)`. HTTP: `fetch(/api/review/diff?cwd=...)` → parse JSON → return `.diff`. Returns raw diff string.

    - `rejectHunk(cwd: string, patchContent: string): Promise<{ success: boolean, error?: string }>` — IPC: `invoke(REVIEW_REJECT_HUNK, cwd, patchContent)`. HTTP: `POST /api/review/reject-hunk` with JSON body.

    - `rejectFile(cwd: string, filePath: string): Promise<{ success: boolean, error?: string }>` — IPC: `invoke(REVIEW_REJECT_FILE, cwd, filePath)`. HTTP: `POST /api/review/reject-file` with JSON body.

    - **Comment state management** (in-memory, per session):
      - Private `comments = new Map<string, ReviewComment[]>()` keyed by sessionId
      - `addComment(sessionId, filename, line, text): ReviewComment` — creates comment with `crypto.randomUUID()` id, adds to map
      - `getComments(sessionId, filename?): ReviewComment[]` — returns all or filtered by filename
      - `toggleResolved(sessionId, commentId): void` — toggles resolved flag
      - `removeComment(sessionId, commentId): void` — removes a comment
      - `clearSession(sessionId): void` — removes all comments for session
      - `getCommentsForFile(sessionId, filename): ReviewComment[]` — convenience method

    - **File review state** (in-memory, per session):
      - Private `fileStates = new Map<string, Map<string, ReviewFileState>>()` keyed by sessionId → filename
      - Methods to get/set hunk states and reviewed flag

    Import from `../../shared/ipc-channels` and `../models/code-review.model`.
