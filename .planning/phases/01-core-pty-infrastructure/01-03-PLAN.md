Create SessionStateService (src/app/services/session-state.service.ts):
- Injectable service (providedIn: 'root')
- Uses RxJS BehaviorSubject for reactive state management
- Interface for active session state:
  ```typescript
  interface ActiveSession {
    metadata: SessionMetadata;
    pid: number;
    scrollbackBuffer: ScrollbackBuffer;
  }
  ```
- Properties:
  - `private sessionsSubject = new BehaviorSubject<Map<string, ActiveSession>>(new Map())`
  - `public sessions$ = this.sessionsSubject.asObservable()`
- Methods:
  1. `addSession(metadata: SessionMetadata, pid: number): void`
     - Create new ActiveSession with fresh ScrollbackBuffer
     - Add to map
     - Emit updated map
  2. `removeSession(sessionId: string): void`
     - Delete from map
     - Emit updated map
  3. `getSession(sessionId: string): ActiveSession | undefined`
     - Return session from map
  4. `appendOutput(sessionId: string, data: string): void`
     - Get session from map
     - Append data to scrollbackBuffer
     - Emit updated map (or use separate subject for output if needed)

Create ScrollbackBuffer class (src/app/services/scrollback-buffer.service.ts):
- NOT an injectable service, just a class
- Circular buffer implementation with fixed size
- Properties:
  - `private buffer: string[] = []`
  - `private readonly maxLines: number`
  - `private head = 0` (write position)
- Constructor: `constructor(maxLines = 10000)`
- Methods:
  1. `append(line: string): void`
     - If buffer not full, push line
     - If buffer full, overwrite oldest line at head position
     - Increment head with modulo: `head = (head + 1) % maxLines`
  2. `getLines(): string[]`
     - If buffer not full, return buffer as-is
     - If buffer full, return in chronological order: `[...buffer.slice(head), ...buffer.slice(0, head)]`
  3. `clear(): void`
     - Reset buffer and head
- Per RESEARCH.md Pattern 5 (Circular Buffer for Scrollback)
