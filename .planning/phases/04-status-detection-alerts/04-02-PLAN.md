Create `src/src/app/services/audio-alert.service.ts`:
- `@Injectable({ providedIn: 'root' })` Angular service
- Private `audioCtx: AudioContext | null = null` — created lazily on first alert call
- Private `muted: boolean` — initialized from `localStorage.getItem('audio-muted') === 'true'`
- Private `debounceTimer: any = null` and `pendingAlert: { status: TerminalStatus; priority: number } | null = null`
- Priority map: ERROR = 3, WAITING = 2, DONE = 1
- `get isMuted(): boolean` returns muted state
- `toggleMute(): void` — toggles muted, saves to `localStorage.setItem('audio-muted', ...)`
- `alert(status: TerminalStatus): void`:
  - If muted, return
  - If priority is 0 (not ERROR/WAITING/DONE), return
  - Debounce: if pendingAlert exists with higher or equal priority, skip. Otherwise update pendingAlert.
  - If no debounceTimer running, start one: `setTimeout(() => { playSound(pendingAlert.status); reset; }, 100)`. After the first sound plays, set a 2-second cooldown debounce timer. During cooldown, accumulate to pendingAlert (highest priority wins). When cooldown fires, play if pending, then reset.
- `playSound(status)`: switch on status, call playWaitingChime / playErrorTone / playDoneChime
- `playWaitingChime(ctx)`: 2-note rising chime — C5 (523.25 Hz, 0.15s) then E5 (659.25 Hz, 0.2s, 0.15s delay). Use 'sine' waveform.
- `playErrorTone(ctx)`: Low buzzy alert — A3 (220 Hz, 0.3s, 0s delay). Use 'sawtooth' waveform.
- `playDoneChime(ctx)`: 3-note descending — G5 (783.99 Hz, 0.12s), E5 (659.25 Hz, 0.12s, 0.12s delay), C5 (523.25 Hz, 0.2s, 0.24s delay). Use 'sine'.
- `playNote(ctx, freq, duration, delay, type)`: Creates OscillatorNode + GainNode. Gain starts at 0.15, ramps down via `exponentialRampToValueAtTime(0.001, ...)`. Connect osc -> gain -> destination. Start and stop with offset. This prevents click artifacts.
- Private `getContext()`: Lazily creates AudioContext. If suspended, call `.resume()`.
