<decisions_made>

- StatusDetector completely rewritten: Original approach (simple regex on ❯ prompt) failed because Claude Code v2.1.56 uses full-screen TUI with constant screen redraws. New approach uses:
  1. OSC window title parsing (✳ Claude Code = idle indicator)
  2. Content hash dedup (ignore identical TUI redraws)
  3. TUI chrome filtering (strip box-drawing chars, known static text)
  4. Only "significant" text (>5 chars after filtering) triggers state changes
  5. INPUT_PATTERNS checked BEFORE WORKING transition (❯, Y/n, Allow once, bypass permissions)
  6. Idle timer: 5s → THINKING, 12s or 5s+✳title → WAITING
- Added process.on('uncaughtException') to prevent node-pty AttachConsole crashes from killing Electron
- Auto-restored sessions now get StatusDetector + processOutput/processExit wiring in main.ts registerRestoredPty()
</decisions_made>
