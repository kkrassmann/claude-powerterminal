2. Create `src/shared/ws-protocol.ts` with shared message type definitions used by both server (Electron main) and client (Angular renderer):

   ```typescript
   // Server -> Client messages
   export type ServerMessage =
     | { type: 'output'; data: string }      // PTY output chunk
     | { type: 'exit'; exitCode: number }     // PTY process exited
     | { type: 'buffering'; total: number }   // Buffer replay starting (line count)
     | { type: 'buffered' }                   // Buffer replay complete

   // Client -> Server messages
   export type ClientMessage =
     | { type: 'input'; data: string }                   // User keyboard input
     | { type: 'resize'; cols: number; rows: number }    // Terminal resize

   // WebSocket close codes (custom range 4000-4999)
   export const WS_CLOSE_CODES = {
     MISSING_SESSION_ID: 4000,
     SESSION_NOT_FOUND: 4004,
     SERVER_SHUTDOWN: 4001,
   } as const;

   export const WS_PORT = 9800;
   ```

   The `buffering`/`buffered` message pair is important: it tells the client when buffer replay starts and ends, so the client can call `terminal.reset()` before replay to avoid duplicate content (Pitfall 3 from RESEARCH.md).
  </action>

<task type="auto">
  <name>Task 2: Create WebSocket server with PTY bridge, scrollback buffer, and heartbeat</name>
  <files>electron/websocket/ws-server.ts, electron/ipc/pty-handlers.ts, electron/main.ts</files>
  <action>
1. Create `electron/websocket/ws-server.ts` implementing the WebSocket server:

   **Imports:** `WebSocketServer`, `WebSocket` from `ws`; `getPtyProcesses` from `../ipc/pty-handlers`; `ScrollbackBuffer` from `../../src/src/app/services/scrollback-buffer.service`; `WS_PORT`, `WS_CLOSE_CODES` from `../../src/shared/ws-protocol`.

   **Module-level state:**
   - `scrollbackBuffers: Map<string, ScrollbackBuffer>` — per-session scrollback buffers
   - `wss: WebSocketServer | null` — server instance reference
   - `heartbeatInterval: NodeJS.Timeout | null` — heartbeat timer reference

   **Export `getScrollbackBuffers()`** — returns the scrollback buffers map. Needed so that when PTY processes are spawned (in pty-handlers.ts), the output can be captured into the scrollback buffer.

   **Export `startWebSocketServer(): WebSocketServer`:**
   - Create `new WebSocketServer({ port: WS_PORT })` and store in `wss`
   - Log: `[WebSocket] Server listening on port ${WS_PORT}`

   **Heartbeat (30-second interval):**
   - Iterate all clients, terminate those with `isAlive === false`, then set `isAlive = false` and send `ping()`
   - On `pong` event, set `isAlive = true`
   - Clear interval on server close

   **Connection handler (`wss.on('connection', ...)`):**
   - Extract `sessionId` from URL path: parse `req.url` to get last segment of `/terminal/{sessionId}`
   - If no sessionId: `ws.close(WS_CLOSE_CODES.MISSING_SESSION_ID, 'Missing sessionId')` and return
   - Look up PTY process via `getPtyProcesses().get(sessionId)`
   - If no PTY: `ws.close(WS_CLOSE_CODES.SESSION_NOT_FOUND, 'Session not found')` and return
   - Set `(ws as any).isAlive = true`
   - Set `(ws as any).sessionId = sessionId` (for debugging)

   **Scrollback buffer replay on connect:**
   - Get or create scrollback buffer from `scrollbackBuffers` map for this sessionId
   - If buffer has content:
     - Send `{ type: 'buffering', total: buffer.getLineCount() }`
     - For each line from `buffer.getLines()`, send `{ type: 'output', data: line }`
     - Send `{ type: 'buffered' }`
   - **Important:** If no buffer exists yet for this session, create one and start capturing. This handles the case where WebSocket connects to a session that was spawned via IPC (Phase 1) before the WS server tracked it.

   **Forward PTY output to WebSocket:**
   - Register `ptyProcess.onData(data => ...)` handler
   - Append data to scrollback buffer: `buffer.append(data)`
   - If `ws.readyState === WebSocket.OPEN`, send `JSON.stringify({ type: 'output', data })`
   - **CRITICAL:** Store the handler reference so it can be cleaned up on ws close. Use a closure variable like `const dataDisposable = ptyProcess.onData(...)`.

   **Handle PTY exit:**
   - Register `ptyProcess.onExit(({ exitCode }) => ...)` handler
   - If `ws.readyState === WebSocket.OPEN`, send `JSON.stringify({ type: 'exit', exitCode })`

   **Handle incoming client messages:**
   - `ws.on('message', (raw) => ...)`: parse JSON, switch on `msg.type`:
     - `'input'`: `ptyProcess.write(msg.data)`
     - `'resize'`: Wrap in try/catch (Pitfall from RESEARCH.md — resize on exited PTY crashes on Windows). Call `ptyProcess.resize(msg.cols, msg.rows)`. Log warning on error.

   **Handle WebSocket close:**
   - `ws.on('close', () => ...)`: Log disconnect. Do NOT dispose PTY or buffer — session persists, only the WebSocket connection closes. Clean up the onData handler reference to prevent memory leaks.

   **Handle WebSocket error:**
   - `ws.on('error', console.error)`

   **Export `stopWebSocketServer(): void`:**
   - Close all client connections with `WS_CLOSE_CODES.SERVER_SHUTDOWN`
   - Close the server via `wss.close()`
   - Clear heartbeat interval
   - Set `wss = null`

2. **Modify `electron/ipc/pty-handlers.ts`:**
   - The existing PTY_SPAWN handler already streams output via IPC (`event.sender.send(PTY_DATA, ...)`). This must ALSO feed the scrollback buffer for WebSocket replay.
   - After the line `ptyProcesses.set(sessionId, ptyProcess);` (line ~88), import and call `getScrollbackBuffers` from `../websocket/ws-server` to get the buffer map.
   - Create a ScrollbackBuffer for the new session: `getScrollbackBuffers().set(sessionId, new ScrollbackBuffer(10000))`
   - In the `ptyProcess.onData` handler (line ~93), add: `const buffer = getScrollbackBuffers().get(sessionId); if (buffer) buffer.append(data);`
   - In the `ptyProcess.onExit` handler (line ~100), add cleanup: `getScrollbackBuffers().delete(sessionId);` (but only if session is truly done, not just disconnected — actually, keep the buffer so reconnecting clients can replay. Only delete when PTY exits.)
   - Add import for `ScrollbackBuffer` from `../../src/src/app/services/scrollback-buffer.service` and `getScrollbackBuffers` from `../websocket/ws-server`.

3. **Modify `electron/main.ts`:**
   - Import `startWebSocketServer` and `stopWebSocketServer` from `./websocket/ws-server`
   - In `app.whenReady()`, call `startWebSocketServer()` BEFORE `createWindow()` (so WS server is ready when renderer connects). Place it after `registerPtyHandlers()` and `registerSessionHandlers()`.
   - In the `will-quit` handler, call `stopWebSocketServer()` BEFORE killing PTY processes (close WebSocket connections first, then kill PTYs).
