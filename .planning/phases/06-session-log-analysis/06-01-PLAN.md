Create `electron/analysis/log-analyzer.ts` with:

**Streaming Parser:**
- Use `readline` on JSONL files, line by line — never load entire file
- `parseJsonlFile(filePath)`: Extract per line: tool_use blocks (count by name), usage tokens (aggregate input/output/cache_read/cache_creation), skill detection (user messages starting with `/`), error detection (tool_result with is_error: true)
- Default limits: max 50 JSONL files, max 20,000 lines per file

**Data Sources:**
- Session JSONL: `~/.claude/projects/*/[ID].jsonl`
- Stats cache: `~/.claude/stats-cache.json`
- History: `~/.claude/history.jsonl`

**readStatsCache():** Read stats-cache.json for daily/model statistics
**parseHistory():** Scan history.jsonl for skill/command patterns

**computeRecommendations(stats):** Rule-based engine:

Praise rules (severity='praise'):
- Task tool >15% → "Starker Subagent-Einsatz!"
- Cache-hit >85% → "Exzellente Context-Wiederverwendung."
- Read >20% → "Guter Read-Before-Write-Workflow."
- Grep+Glob > Bash searches → "Saubere Tool-Nutzung."
- GSD skills used → "Orchestrator-Skills im Einsatz!"
- EnterPlanMode used → "Plan-Mode wird aktiv genutzt."

Improvement rules:
- Cache-hit <70% → warning: "Context wird häufig neu aufgebaut."
- Task never used → info: "Subagents nicht genutzt."
- Bash with grep/find/rg/cat >20% → warning: "Bash für Datei-Suche."
- Error rate >10% → warning: "Hohe Fehlerrate."
- No skills → info: "Keine Slash-Commands."
- Read <5% but Write/Edit >20% → warning: "Wenig Read vor Write."
- GSD without verify → info: "GSD ohne Verifikation."
- Sessions >500 messages → warning: "Sehr lange Sessions."
- Orchestrator skills as simple commands → info

**computeSessionScore(sessionPath):** Per-session 0-100 score:
- Tool-Nativeness: 25% (Grep/Glob/Read vs Bash equivalents)
- Subagent-Nutzung: 20% (Task tool share)
- Read-before-Write: 20% (Read vs Write/Edit ratio)
- Context-Effizienz: 20% (Cache-hit ratio)
- Error-Rate: 15% (inverse error rate)

Badges: "Subagent Pro" (Task>15%), "Tool Native" (no bash workarounds), "Context Efficient" (cache>85%), "Planned" (EnterPlanMode), "Orchestrated" (orchestrator skills)

**analyzeAllSessions():** Main entry point, returns SessionAnalysis. 5-minute cache.
