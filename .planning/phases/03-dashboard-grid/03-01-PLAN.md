    1. Create `src/src/app/models/git-context.model.ts`:
       - Export `GitContext` interface with fields: `branch: string | null`, `added: number`, `modified: number`, `deleted: number`, `isGitRepo: boolean`
       - Keep it minimal, matching the pattern in session.model.ts

    2. Create `electron/ipc/git-handlers.ts`:
       - Import `ipcMain` from electron, `execFile` from child_process, `promisify` from util
       - Define same `GitContext` interface (main process can't import from Angular src — duplicate is fine for 5 fields)
       - Export `registerGitHandlers()` function that registers two IPC handlers:

       a) `git:context` handler (takes `cwd: string`, returns `GitContext`):
          - Run `git rev-parse --abbrev-ref HEAD` and `git status --porcelain` in parallel using `Promise.all`
          - Set `timeout: 5000` on both `execFileAsync` calls to prevent hangs on large repos
          - Parse porcelain output: lines with `?` or `A` in XY columns → added, `M` → modified, `D` → deleted
          - On any error (not a git repo, git not installed, timeout), return `{ branch: null, added: 0, modified: 0, deleted: 0, isGitRepo: false }`
          - Important: Use `execFile` (not `exec`) — no shell injection risk, lower overhead

       b) `app:home-dir` handler (no args, returns `string`):
          - Return `process.env.HOME || process.env.USERPROFILE || ''`
          - This gives the renderer the home directory for path shortening (renderer is sandboxed, can't read env vars)

    3. Add IPC channel constants to `src/shared/ipc-channels.ts`:
       - Add `GIT_CONTEXT: 'git:context'` in a new "Git context channels" section
       - Add `APP_HOME_DIR: 'app:home-dir'` in a new "App info channels" section

    4. Register git handlers in `electron/main.ts`:
       - Import `registerGitHandlers` from `./ipc/git-handlers`
       - Call `registerGitHandlers()` in the `app.whenReady().then()` block, alongside existing `registerPtyHandlers()` and `registerSessionHandlers()` calls
  </action>

  <action>
    Create `src/src/app/services/git-context.service.ts`:

    - Injectable with `providedIn: 'root'`
    - Implements `OnDestroy` for cleanup

    **State:**
    - `private contexts = new BehaviorSubject<Map<string, GitContext>>(new Map())`
    - `public contexts$ = this.contexts.asObservable()` — components subscribe to this
    - `private pollInterval: ReturnType<typeof setInterval> | null = null`
    - `private trackedSessions = new Map<string, string>()` — sessionId → cwd mapping
    - `private previousCounts = new Map<string, string>()` — sessionId → serialized counts for change detection
    - `public changedSessions = new Set<string>()` — sessions where counts changed on last poll (for highlight animation)

    **Public methods:**
    - `startPolling()`: If not already polling, call `pollAll()` immediately, then `setInterval(pollAll, 30000)`. Use IPC_CHANNELS.GIT_CONTEXT constant.
    - `stopPolling()`: Clear interval
    - `trackSession(sessionId: string, cwd: string)`: Add to trackedSessions map, trigger immediate poll for that session
    - `untrackSession(sessionId: string)`: Remove from trackedSessions and contexts maps
    - `getContext(sessionId: string): GitContext | undefined`: Synchronous getter from current BehaviorSubject value

    **Private methods:**
    - `async pollAll()`: For each tracked session, call `window.electronAPI.invoke(IPC_CHANNELS.GIT_CONTEXT, cwd)`. Update contexts map. Detect count changes by comparing `${added}:${modified}:${deleted}` to previousCounts — if different, add sessionId to changedSessions. After all polls complete, emit new Map to BehaviorSubject. Clear changedSessions after 1500ms (animation duration).

    **Cleanup:**
    - `ngOnDestroy()`: Call stopPolling()

    **Important patterns (from existing codebase):**
    - Use `window.electronAPI.invoke()` for IPC (same as SessionManagerService)
    - Import `IPC_CHANNELS` from `../../../shared/ipc-channels`
    - Import `GitContext` from `../models/git-context.model`
    - Silent failure on individual session poll errors (keep previous value, log warning)
  </action>
