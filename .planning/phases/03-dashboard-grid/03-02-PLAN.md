    **Step 1: Create DashboardComponent (`src/src/app/components/dashboard/`)**

    `dashboard.component.ts`:
    - Standalone component, selector `app-dashboard`
    - Imports: CommonModule, DragDropModule (from @angular/cdk/drag-drop), TerminalComponent, TileHeaderComponent
    - Inject: SessionStateService, GitContextService, SessionManagerService, PtyManagerService, NgZone
    - State:
      - `sessions: ActiveSession[] = []` — from SessionStateService subscription
      - `pendingSessions: SessionMetadata[] = []` — @Input from parent for resume placeholders
      - `maximizedSessionId: string | null = null`
      - `homeDir: string = ''` — fetched once via IPC on init
    - `ngOnInit()`:
      - Subscribe to `sessionStateService.sessions$` → update `sessions` array, remove pending sessions that became active
      - Fetch home directory: `window.electronAPI.invoke(IPC_CHANNELS.APP_HOME_DIR)` → store in `homeDir`
      - Start git context polling: `gitContextService.startPolling()`
      - Track all existing sessions in gitContextService
    - `ngOnDestroy()`: Unsubscribe, stop polling
    - `toggleMaximize(sessionId: string)`: Toggle `maximizedSessionId`
    - `onDrop(event: CdkDragDrop<ActiveSession[]>)`: Use `moveItemInArray` to reorder sessions
    - `trackBySessionId(index: number, session: ActiveSession): string` — return sessionId
    - `getSession(sessionId: string): ActiveSession | undefined` — find in sessions array
    - `onSessionExited(sessionId: string)`: Emit via @Output to parent
    - `restartSession(sessionId: string)`: Forward to terminal component (or directly invoke PTY_RESTART IPC). Actually — the terminal component already handles restart/kill internally. The tile header buttons need to trigger methods on the terminal. Solution: Use the same IPC approach as TerminalComponent — invoke `PTY_RESTART` and `PTY_KILL` directly.
    - `killSession(sessionId: string)`: Invoke `PTY_KILL` via IPC, then emit sessionExited
    - When sessions change, track/untrack in GitContextService

    `dashboard.component.html`:
    ```
    <!-- Grid view -->
    <div class="dashboard-grid"
         *ngIf="!maximizedSessionId"
         cdkDropList
         cdkDropListOrientation="mixed"
         [cdkDropListData]="sessions"
         (cdkDropListDropped)="onDrop($event)">

      <div *ngFor="let session of sessions; trackBy: trackBySessionId"
           class="tile" cdkDrag>
        <!-- Use cdkDragHandle on tile-header to restrict drag to header only -->
        <app-tile-header
          [session]="session"
          [gitContext]="gitContextService.getContext(session.metadata.sessionId)"
          [isChanged]="gitContextService.changedSessions.has(session.metadata.sessionId)"
          [isMaximized]="false"
          [homeDir]="homeDir"
          (maximize)="toggleMaximize(session.metadata.sessionId)"
          (restart)="restartSession(session.metadata.sessionId)"
          (kill)="killSession(session.metadata.sessionId)"
          cdkDragHandle>
        </app-tile-header>
        <app-terminal
          [sessionId]="session.metadata.sessionId"
          (sessionExited)="onSessionExited($event)">
        </app-terminal>
      </div>

      <!-- Pending resume placeholders -->
      <div *ngFor="let pending of pendingSessions" class="tile pending-tile">
        <div class="pending-indicator">
          <div class="spinner"></div>
          <span class="pending-text">Resuming session</span>
          <code class="pending-id">{{ pending.sessionId }}</code>
          <span class="pending-dir">{{ pending.workingDirectory }}</span>
        </div>
      </div>

      <!-- Empty state -->
      <div *ngIf="sessions.length === 0 && pendingSessions.length === 0" class="no-sessions">
        <p>No active sessions. Create one above to get started.</p>
      </div>
    </div>

    <!-- Maximized view -->
    <div class="maximized-view" *ngIf="maximizedSessionId">
      <app-tile-header
        [session]="getSession(maximizedSessionId)!"
        [gitContext]="gitContextService.getContext(maximizedSessionId)"
        [isChanged]="gitContextService.changedSessions.has(maximizedSessionId)"
        [isMaximized]="true"
        [homeDir]="homeDir"
        (maximize)="toggleMaximize(maximizedSessionId)"
        (restart)="restartSession(maximizedSessionId)"
        (kill)="killSession(maximizedSessionId)">
      </app-tile-header>
      <app-terminal
        [sessionId]="maximizedSessionId"
        (sessionExited)="onSessionExited($event)">
      </app-terminal>
    </div>
    ```

    **Important considerations from research:**
    - Use `cdkDropListOrientation="mixed"` — required for CSS Grid wrapping layouts (CDK 17+)
    - `cdkDragHandle` on the tile-header restricts drag to header only, keeping terminal interactive
    - Maximize uses *ngIf to swap views. This WILL destroy/recreate terminal components. However, the terminal component already handles WebSocket reconnection on init. The research flagged this as Pitfall 7, but the existing reconnect logic handles it. If this causes issues, can be changed to CSS hide/show later.
    - Use a `*cdkDragPreview` with a simplified preview (header + colored rectangle) to avoid canvas cloning issues flagged in research Open Question 1
