  <action>
Add automatic buffer resync every 30 seconds for remote browsers:

1. Add a private property: `private resyncInterval?: any;`

2. In `ngOnInit()` or in the WebSocket connection success callback, start the interval (only for remote browsers):
   ```typescript
   // Periodic buffer resync for remote browsers (prevents xterm.js desync)
   if (!window.electronAPI && this.ws && this.ws.readyState === WebSocket.OPEN) {
     this.resyncInterval = setInterval(() => {
       if (this.ws && this.ws.readyState === WebSocket.OPEN && this.session?.id) {
         // Request buffer replay
         this.ws.send(JSON.stringify({
           type: 'buffer-replay',
           sessionId: this.session.id
         }));
       }
     }, 30000); // 30-second interval
   }
   ```

3. Add handler for 'buffer-clear' and 'buffer-replay' messages in the `ws.onmessage` handler:
   ```typescript
   if (message.type === 'buffer-clear') {
     this.terminal?.clear();
   } else if (message.type === 'buffer-replay') {
     this.terminal?.clear();
     this.terminal?.write(message.data);
   }
   ```

4. In `ngOnDestroy()`, clear the interval: `if (this.resyncInterval) { clearInterval(this.resyncInterval); }`

**Why this fixes the gap:** User reported glitches resolve with a "refresh" — this automates that refresh every 30 seconds for remote browsers only.

**Why remote-only:** Electron browsers use the same machine's PTY with lower latency and no network issues. Remote browsers are more prone to network-induced desync.
